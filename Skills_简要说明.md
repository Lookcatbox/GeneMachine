# 技能库简要说明（中文）

以下列出可用技能及其中文简要说明，便于在项目中查阅。

- microsoft-foundry：用于在 Microsoft Foundry 平台上部署、管理、评估和优化智能代理与模型的端到端工作流工具集说明与操作建议。
- caveman：超压缩沟通模式，生成极简短、技术精确的回答以节省代币与阅读时间。
- diagnose：系统化故障诊断流程（复现→最小化→假设→检测→修复→回归测试）的指导与执行步骤。
- edit-article：对文章进行结构改写、语言精简与可读性提升的编辑辅助工具。
- git-guardrails-claude-code：为仓库设置防止破坏性 Git 操作（如强制推送、硬重置等）的保护钩子建议与实现。
- grill-me：对方案或设计进行严格质询，帮助用户充分检验与完善设计决策的交互式技巧。
- grill-with-docs：在现有文档与领域模型基础上，对方案进行更严格的质询并同步更新相关文档（如 CONTEXT.md、ADR）。
- handoff：将当前工作浓缩为交接文档，便于其他工程师或代理继续处理。
- improve-codebase-architecture：分析代码库架构并提出重构、可测性、模块化与可维护性的改进建议。
- migrate-to-shoehorn：将测试中使用的 `as` 类型断言迁移为 @total-typescript/shoehorn 的实用迁移工具与建议。
- obsidian-vault：在 Obsidian 笔记库中搜索、创建与管理笔记（含 wikilinks 与索引），用于知识管理。
- prototype：快速搭建可运行的原型（终端应用或多方案 UI），用于验证设计与业务逻辑假设。
- review：对变更集（自某提交点起）进行双维度审查：代码规范与需求吻合度评估。
- scaffold-exercises：脚手架化练习题目录结构（题目、答案、解析），并确保通过 lint 与格式检查。
- setup-pre-commit：为仓库配置 Husky + lint-staged（Prettier）以及提交时的检查流程（类型检查、测试等）。
- tdd：引导红绿重构的测试驱动开发循环，生成测试用例与实现建议。
- to-issues：将计划或 PRD 拆分为可独立拿取的 issue（垂直切片），便于任务分配与跟踪。
- to-prd：把会话上下文整理为产品需求文档并发布为项目 issue。
- triage：问题分诊流程与状态机管理，帮助准备与优先级划分 incoming issues。
- write-a-skill：创建或编辑代理技能文件（.instructions.md、.agent.md、SKILL.md 等）的规范化流程与模版。
- writing-beats：按段落节奏构建文章，每步呈现可选方向并逐段生成内容。
- writing-fragments：采集用户碎片化写作素材（短句、片段），归档到单一文档供后续编辑使用。
- writing-shape：将原始素材逐步整理成文章，通过交互式会话构建最终稿。
- project-setup-info-local：为本地 VS Code 工作区生成完整的项目初始化与脚手架步骤（适用于多种框架）。
- get-search-view-results：从 VS Code 的 Search 视图中获取当前搜索结果的工具。
- agent-customization：修改或调试 VS Code 代理定制文件（.instructions.md、.prompt.md 等）的工作流与修复建议。
- vscode-microsoft-foundry：针对 Microsoft Foundry 平台的端到端代理应用开发工作流与实践指南。
- foundrytk-quick-start：Foundry Toolkit 的快速入门指南，帮助用户设置 Foundry 并快速构建/测试代理。

> 文件生成于项目根目录，供团队查阅与快速理解各项技能用途。
